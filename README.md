# Modular App Frameworks

Учебный проект по дисциплине **Технологии разработки приложений на базе фреймворков**  
Цель: построение расширяемого монолитного приложения с модульной архитектурой и контейнером внедрения зависимостей.

---

## Описание

Приложение реализует ядро, которое загружает модули, учитывает их зависимости, регистрирует службы через контейнер внедрения зависимостей и выполняет действия модулей.  
Основные особенности:

- Модули независимы от ядра, расширяемость без изменения логики ядра.
- Порядок запуска модулей определяется с учётом зависимостей.
- Проверка ошибок: отсутствующие модули и циклические зависимости.
- Контейнер внедрения зависимостей управляет временем жизни объектов (singleton и transient).
- Поддержка тестирования модулей и зависимостей.

## Установка и запуск

1. Клонируем репозиторий:

```bash
git clone https://github.com/Kuonkli/modular_app_frameworks.git
cd modular_app_frameworks
````

2. Запуск приложения:

```bash
go run ./cmd
```

Приложение загружает модули из `config/modules.json`, строит порядок запуска по зависимостям, регистрирует службы в контейнере и выполняет действия модулей.

---

## Структура модулей

Каждый модуль имеет:

- `name` — имя модуля
    
- `requires` — список зависимостей (имен модулей)
    
- `register(container)` — регистрация служб в контейнере
    
- `init(container)` — инициализация модуля после регистрации

Пример зависимостей:

```
Core       -> не требует других
Validation -> требует Core
Export     -> требует Core, Validation
Report     -> требует Core, Export
Logging    -> требует Core
```

---

## Контейнер внедрения зависимостей

- `singleton` — создаётся один раз, повторно используется
    
- `transient` — создаётся каждый раз при запросе
    
- Доступ к службам через `container.Get("serviceName")`
    
- Позволяет легко внедрять зависимости в тестах или модулях

---

## Тесты

Примеры тестов:

- Проверка корректного порядка запуска модулей
    
- Проверка ошибки при отсутствии модуля
    
- Проверка ошибки циклических зависимостей
    
- Проверка, что зависимости действительно внедряются через контейнер

Запуск тестов:

```bash
go test ./tests/...
```

## Демонстрация

```

Действие Экспорт данных в файл
Действие Формирование отчёта
Действие Проверка журнала событий
Файл экспорта создан
```

Файл `export.txt` создаётся в рабочей директории.

---

## Как добавить новый модуль

1. Создать файл в `modules/`
    
2. Описать имя, зависимости, методы `register` и `init`
    
3. Добавить имя модуля в `config/modules.json`
    
4. Ядро не требует изменений — новый модуль автоматически учитывается
